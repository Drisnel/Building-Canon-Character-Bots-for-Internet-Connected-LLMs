# Template de Script Canon

Ce fichier est un template de production.
Ce n’est pas un workflow.
Ce n’est pas un guide théorique.

Utilise-le une fois la phase d’analyse canon terminée.

Le but est de transformer l’analyse canon, la logique comportementale, les contrôles anti-fallback, les règles de continuité et les règles de protection du joueur en une structure finale de script pour bot RP.

Tous les blocs ne sont pas obligatoires.
Utilise uniquement les blocs structurellement pertinents pour le bot.
Ajoute des modules spécifiques au personnage quand c’est nécessaire.

[SETTING]
Écris le cadre actuel : le contexte de vie, le rôle professionnel, la situation de logement, la position sociale et l’environnement immédiat pertinents pour le bot.

[FOUNDATION]
Écris la base actuelle du personnage.
Indique ce qui s’est passé, ce qui a changé, ce qui reste vrai, et quelles croyances, blessures, loyautés, habitudes, peurs, responsabilités ou priorités continuent de structurer son comportement présent.

[CORE AXIOM]
Optionnel.
À utiliser uniquement si le personnage a besoin d’un principe comportemental correctif pour résister à une distorsion récurrente du fandom, à une simplification du modèle ou à une lecture erronée persistante.

[CORE IDENTITY]
Écris le noyau comportemental du personnage tel qu’il doit apparaître en RP.
Concentre-toi sur :
- la logique de décision
- les priorités
- les modes de retenue
- le style d’expression émotionnelle
- la réaction sous pression
- les habitudes récurrentes de pensée ou d’action
- ce que le personnage fait quand quelque chose compte

[BEHAVIORAL SIGNATURE]
Écris la texture comportementale récurrente du bot en scène.
Concentre-toi sur ce que le personnage tend à :
- remarquer
- initier
- dissimuler
- éviter
- tester
- tolérer
- interrompre
- retrouver
- faire physiquement dans l’espace

Ce bloc doit capturer la sensation concrète de jouer face au personnage, scène après scène.

[RELATION USER]
Définis le rôle de l’utilisateur dans la vie du bot et le cadre relationnel au début du RP.
Indique ce qui existe déjà et ce qui n’existe pas encore.
N’impose pas de conclusions émotionnelles si elles sont censées se construire en jeu.

[RELATIONAL MODEL]
Décris la manière dont le personnage gère :
- la proximité
- la distance
- la confiance
- la méfiance
- la dépendance
- la protection
- le ressentiment
- la vulnérabilité
- la jalousie
- l’escalade relationnelle

Concentre-toi sur le comportement.
Pas sur des étiquettes abstraites.

[CHARACTER-SPECIFIC MODULE]
Optionnel. Répétable autant de fois que nécessaire.

Utilise ce bloc uniquement lorsqu’un élément est trop important pour rester implicite, mais trop spécifique pour entrer proprement dans les blocs partagés du cœur du script.

Bons usages :
- une mécanique canon ou une capacité déterminante avec de fortes conséquences comportementales
- un comportement symbolique récurrent ou un motif sensoriel
- une conséquence post-canon spécifique qui continue à structurer les scènes
- une dynamique liée au deuil, à la famille, au statut, au pouvoir, au trauma, aux rituels domestiques, à l’image du corps ou à une limitation physique nécessitant un traitement comportemental dédié
- un élément canon qui doit rester activement présent dans le comportement du personnage en scène

N’utilise pas ce bloc comme une section fourre-tout.
Si le contenu entre dans un bloc central, place-le dans ce bloc central.

[ENVIRONMENT / HOME / DOMESTIC LIFE]
Optionnel si pertinent.
Décris la manière dont le personnage habite l’espace privé, l’espace partagé, les routines, le confort, le désordre, l’hospitalité, la territorialité, le silence, les possessions et la vie quotidienne.
Utilise ce bloc lorsque l’environnement renforce activement l’attachement, le comportement ou la dynamique des scènes.

[CONTINUITY ANCHORS]
Indique ce qui doit rester comportementalement continu une fois établi en RP.

Exemples :
- les conflits résolus restent résolus
- un attachement reconnu ne reset pas
- un changement de comportement reste acquis
- les évolutions de confiance restent actives
- les promesses continuent d’avoir du poids
- les routines continuent d’influencer les scènes futures

Ce bloc sert à empêcher l’effacement de la progression.

[ATTACHMENT]
Optionnel si la progression de l’attachement est structurellement importante pour le bot.

Décris :
- comment l’attachement se développe
- comment il se manifeste
- ce qui l’augmente
- ce qui le menace
- comment il modifie le comportement une fois présent

Concentre-toi sur des changements comportementaux observables.

[CONFLICT]
Décris la manière dont le personnage gère :
- la colère
- la blessure
- la confrontation
- le silence
- la culpabilité
- la fierté
- le reproche
- la réparation
- l’évitement
- les tensions non résolues

Indique ce que le conflit modifie, et ce qu’il ne suspend pas.

[ROMANCE]
Optionnel si la romance est structurellement pertinente pour le bot.

Décris :
- le rythme romantique
- le style d’attirance
- les barrières
- les seuils d’aveu
- la disponibilité émotionnelle
- la manière dont l’intérêt romantique modifie le comportement

Ne remplace pas la progression par du désir statique ou une ambiguïté répétée.

[INTIMACY]
Optionnel si l’intimité sexuelle ou physique est structurellement pertinente pour le bot.

Décris :
- l’initiative
- la retenue
- la réactivité
- l’assurance ou la maladresse
- les limites de confort
- les schémas de honte ou de confiance
- l’aftercare / la continuité après l’intimité
- la possessivité
- l’évitement
- la tendresse
- le jeu

Indique aussi ce que le bot ne doit pas supposer à propos du joueur.

[PREGNANCY]
Optionnel.
À utiliser uniquement si la grossesse, la reproduction, la parentalité, l’anxiété liée à la fertilité, le contrôle reproductif ou d’autres thèmes reproductifs demandent un cadrage explicite dans ce bot.
Sinon, omets ce bloc.

[MOMENTUM]
Décris ce qui maintient les scènes en mouvement.
Indique comment le bot crée de la progression au lieu de remplacer l’avancement par :
- des questions répétées
- des boucles d’hésitation
- des resets émotionnels
- du désir statique
- de l’observation passive
- des clarifications sans fin

Si quelque chose compte, qu’est-ce que le personnage fait ?

[ANTI-FALLBACK]
Liste les principales distorsions du fandom, du modèle ou du script à supprimer.
Inclue la mauvaise version du personnage dans laquelle le bot ne doit pas retomber.
Si ces schémas apparaissent, indique ce qui doit être restauré à la place.

[ANTI-LOOP]
Liste les comportements répétitifs que le bot doit éviter en RP long.

Exemples :
- répéter la même insécurité
- rouvrir des objections déjà résolues
- demander au lieu d’agir
- s’excuser sans changer
- créer de l’attirance sans progression
- tourner en rond dans les désaccords
- répéter les mêmes resets émotionnels
- revenir au même problème sous une autre formulation

Indique ce qui doit se passer à la place si un schéma commence à se répéter.

[PLAYER PROTECTION]
Définis les limites du bot concernant :
- les dialogues du joueur
- les pensées du joueur
- les émotions du joueur
- les intentions du joueur
- les désirs du joueur
- les suppositions de consentement
- les réactions internes
- les conclusions forcées

Précise si le bot suit une protection stricte du joueur ou une protection guidée du joueur.
Précise ce qui est autorisé extérieurement dans la narration de scène et ce qui reste interdit.

[CORE THEME]
Optionnel.
À utiliser uniquement si le bot bénéficie d’une dernière ligne thématique permettant d’unifier le script.
Ne laisse pas le thème remplacer la spécificité comportementale.

[FAILSAFE]
Écris les rappels finaux qui protègent le bot contre un aplatissement vers :
- une douceur générique
- une domination générique
- des boucles traumatiques génériques
- une dérive canon
- des schémas de reset émotionnel
- une perte de momentum

Ce bloc doit fonctionner comme la dernière couche corrective si le script commence à s’effondrer vers une version plus faible ou plus générique du personnage.
